# Support Policy

This document explains how to get help and what response expectations to assume.

## Where to Ask

- **Bug reports / feature requests**: GitHub Issues
- **Security concerns**: follow `SECURITY.md` (private disclosure path)
- **Usage and onboarding questions**: GitHub Discussions or Issues (when Discussions are not enabled)

## What to Include

For faster triage, include:

- Synapse version (SDK/package/service version)
- Runtime details (Python/Node/OS)
- Reproduction steps
- Expected behavior vs actual behavior
- Logs/screenshots (for UI or runtime issues)

## Support Scope

- OSS support is best-effort and community-driven.
- Priority is given to core loop reliability and documented features.
- Enterprise SLAs are out of scope for this public repository.

## Response Targets (Best Effort)

- New bug triage: within 5 business days
- Security issue acknowledgment: within 2 business days
- Feature request triage: within 10 business days

No guaranteed resolution time is provided for OSS issues.

## Supported Baselines

See `docs/compatibility-matrix.md` for tested runtime versions and support signals.
