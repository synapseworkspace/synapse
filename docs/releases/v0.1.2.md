# Synapse v0.1.2

Release date: 2026-04-04

This release closes two critical execution items for production adoption:
- adds automated pre-prod chaos/recovery verification for self-hosted deployments;
- introduces low-latency Postgres external-memory ingestion (`wal_cdc`) on top of existing `postgres_sql` polling mode;
- finalizes wiki-first core UX recovery work so first-run operators land in wiki workflows, not control-plane overload.

## Notes

- `postgres_sql` now supports both `sql_sync_mode=polling` and `sql_sync_mode=wal_cdc`; polling remains default for backward compatibility.
- Self-hosted chaos drill is opt-in in CI via workflow dispatch input `run_selfhost_chaos_drill=true`.
